Switch Back to VS Code

2022-01-04 :: Dery R Ahaddienata
#tech  #editor  #vscode 

It’s about 1.5 year I use vim as my default IDE. But, recently I found reduction in term of productivity when using vim for heavy-tools coding task. So, here, I make a simple comparision between VS Code and Vim, how one has advantages than other.

Comparison⌗

Vim / NeovimVS Code
Typing SpeedReally fast (+)Just so-so (.)
ConfigurationHighly configurable (+) but it’s hard & takes so much time (-)Built-in, support many plugin and has more standard plugin configuration than vim (+)
Widely UseGithub star 49.8k (nvim), 25.7k (vim)Github star 126k

PerformanceAmazingly powerful (+), but to make vim support modern IDE, installing many plugin make the performance even worst (-)Still powerful, I believe VS Code might leverage the resource more efficient than vim (+)

Analyze⌗

I felt the open source project is still the best but today’s open source need to have backup from gigantic tech company to support their maintainer. VS Code is under microsoft, I believe it’s more sustain than Vim / NVim. Vim configuration is not so standard, hard to maintain, even NVim provide more realible configuration format, but it’s still hard to introduce new config. If you use vim to develop vim, using vim is totally fine, but if you use vim to develop other app, I would not recommend, it takes lot of time, and most of the time, with dynamic tools like we face today, using vim means you will learn 80% vim and 20% code, deployment, etc. Since vim requires extra mental aritmatic model, we will lost the important of learning how to build software itself. Also compared with vscode, vscode has advantages when you need to move to another company / techstack / even pairing with others.

Beside those limitation, I believe if you’re non novice programmer and have good balance of memorization + aritmetic, you can utilize vim as your productivity tools. If you can takes no-time to configure vim or you know how vim works behind the scene, you can use it. If most of the time, when you’re using vim to code, and take only small amount of time learning vim, let say 5%, the rest 95% can be allocated into the coding activity itself.

Conclusion⌗

So, I decide to use VS Code + Vim interchangebly. I use VS Code for heavy software development task, and use Vim as my go to editing software. By this, I can leverage both advantages. ðŸ”Œ
