I’ve been using vim as my default IDE for about a year. Until now I’m still improving the vim configuration so that I could do the task even more effective. Now after nvim launch version 0.5.0 which support native LSP client, I immediately tried this feature. Yeps, there’re no significant difference when I use LanguageClient-neovim, but since version 0.5.0 has built in feature for LSP client, I would preferably use the built in one.

If we compare with my last setup, we can see significant config differences. And because I also planned to try contribute on LunarVim project, I would like to documenting all of the setup I used now.

    function GetSessionFileName() abort
      if len(argv()) > 0
        return substitute(fnamemodify(argv()[0], ':p:h'), "/", "_", "g")
      endif
      return substitute(fnamemodify(getcwd(), ':p:h'), "/", "_", "g")
    endfunction!

    let g:session_path = join(['~/.vim/sessions/', GetSessionFileName(), '.vim'], '')

    " Save session on quitting Vim
    autocmd VimLeave * NERDTreeClose
    autocmd VimLeave * execute 'mksession!' g:session_path

    " Restore session function
    function RestoreSession() abort
        execute 'source' g:session_path
        autocmd VimEnter * NERDTree
    endfunction!

    map <leader>r  :call RestoreSession()<CR>
